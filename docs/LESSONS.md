# 教訓・自己改善ルール

> **セッション開始時に「再発防止ルール」を必ず確認してから作業を始める。**  
> **ユーザーから修正・指摘を受けたら、作業再開前にこのファイルに記録する。**

---

## 再発防止ルール（クイックリファレンス）

*毎セッション開始時に読む。ミスの度に行を追加する。*

| # | ミスパターン | 防止ルール |
|---|------------|----------|
| 1 | 新機能追加後にドキュメントが更新されず、関数名・ファイル構成・チャート数が実装と乖離する | 実装変更後は `/docs-sync` を実行してから PR を出す |
| 2 | デバッグ例のコードに存在しない関数名（例: `calculateCompound`）を書く | デバッグ例を書く前に `script.js` の関数名を Grep で確認する |
| 3 | タブ・シミュレーターが増えてもドキュメントの「現在は〇個」という記述を更新し忘れる | 数を明記した文言を書くときは「現在〇個（要更新）」と注記するか、数を書かない |

## 詳細記録

「なぜ起きたか・なぜそうしたか・次回どうするか」にフォーカスして書く。

---

### 2026-05-19: ドキュメントと実装の乖離（3シミュレーター追加後）

**症状**: `calculateCompound()` というデバッグ例が CLAUDE.md / DEVELOPMENT.md に残っていたが、実際の関数名は `buildYearlyData()`。また「canvas 要素を複数持たない、折れ線グラフ 1 個」という記述が残っていたが、実際は `chart` / `dividendChart` / `retirementChart` の 3 つが存在した。

**根本原因**: 配当シミュレーター・老後資金シミュレーターを追加したときに、ドキュメント更新チェックが抜けた。

**再発防止ルール**: 再発防止ルール表の #1 に記録済み。実装変更後は `/docs-sync` を実行する。

---

### 2026-05-12: アーキテクチャは固定ではないことを明示する重要性

**症状**: ドキュメントに「現在の構成」とだけ書くと、AI エージェントや新人開発者が「それが永続的な決定」と誤解する可能性がある。

**再発防止ルール**: アーキテクチャ関連の記述には「今後変わる可能性あり」という注記を添える。変更時は CLAUDE.md / core.md / AGENTS.md / DEVELOPMENT.md の 4 ファイルをセットで更新する。

---

### 2026-05-12: ビルドプロセスなしの構成のメリットを過小評価しない

**症状**: 「フレームワーク化すれば〇〇が楽」という提案が出やすいが、現在の Vanilla JS 構成は保守コストが低い。

**再発防止ルール**: フレームワーク化・ビルドツール導入を提案する前に、オンボーディング時間・ビルド保守コスト・バンドルサイズのトレードオフをコスト計算に含める。必ずユーザーに相談してから進める。

---

### 2026-05-12: Chart.js CDN 読み込み順序の問題

**症状**: グラフが表示されず、コンソールエラーなし。

**根本原因**: CDN スクリプトに `async` 属性を付けたため、`script.js` が Chart.js より先に実行された。

**再発防止ルール**: CDN に依存するスクリプトは `async` を付けない。読み込み順を明示的に制御する。

```html
<!-- ❌ NG -->
<script src="https://cdn.jsdelivr.net/npm/chart.js" async></script>
<script src="script.js"></script>

<!-- ✅ OK -->
<script src="https://cdn.jsdelivr.net/npm/chart.js"></script>
<script src="script.js"></script>
```
