ユーザーから受けた修正・指摘、またはセッション中に気づいた教訓を `docs/LESSONS.md` に記録してください。

## 手順

1. `docs/LESSONS.md` を読む
2. 記録すべき内容を以下の観点で特定する:
   - ユーザーから受けた修正・指摘（最優先）
   - 実装中に発見した非自明な問題
   - 次回以降のセッションで繰り返したくない判断ミス

3. **再発防止ルール表**（クイックリファレンス）に1行追加する
4. **詳細記録**セクションに詳細を追記する

## 追記フォーマット

### 再発防止ルール表（先頭テーブル）に追加

```markdown
| N | ミスパターンを短く説明 | 次回どうするかを1行で |
```

### 詳細記録セクションに追加

```markdown
### YYYY-MM-DD: [ミス・教訓のタイトル]

**症状**: 何が起きたか・何を指摘されたか

**根本原因**: なぜ起きたか

**再発防止ルール**: 再発防止ルール表の #N に記録済み。具体的な行動ルールを書く。

（コードスニペットがあれば ✅ OK / ❌ NG パターンで示す）
```

## 注意事項

- 「何をしたか」ではなく「なぜ起きたか・次回どうするか」にフォーカス
- 再発防止ルール表は**毎セッション開始時に読まれる**ので、短く・実行可能な形で書く
- 今日の日付（YYYY-MM-DD 形式）を使う
- 同じパターンの繰り返しなら、既存の行を「強化」する（重複追加しない）
