---

FundFlow のドキュメント整合性を確認してください。以下の手順で実施します。

## チェック対象ファイル

- `CLAUDE.md` — 技術スタック・ファイル構成・計算ロジック・デバッグ例
- `.claude/rules/core.md` — ファイル配置ルール・グラフ描画ルール
- `.agents/AGENTS.md` — よくある質問・よくあるハマりどころ
- `docs/DEVELOPMENT.md` — ファイル構成・デバッグ例・コマンド

## チェック項目

### 1. 関数名の整合性

`script.js` の実際の関数名（`function ` で始まる行）を確認し、ドキュメントに記載された関数名と照合する。

確認ポイント:
- デバッグ例のコードブロックに存在しない関数名が使われていないか
- よくあるハマりどころのコード例が実際のコードと一致しているか

### 2. チャート数の整合性

`script.js` で `let.*Chart` または `new Chart(` の件数を確認し、ドキュメントの記述と照合する。

### 3. ファイル構成の整合性

実際に存在するファイル一覧（index.html / script.js / style.css / manifest.json / icons/ など）と、各ドキュメントのファイル構成表を照合する。

### 4. テック スタックの整合性

`index.html` の Chart.js バージョン（CDNのURLから）を確認し、CLAUDE.md の「Chart.js 4（CDN）」記述と照合する。

### 5. コマンドの整合性

CLAUDE.md と DEVELOPMENT.md のセットアップコマンドが一致しているか確認する。

## 出力形式

```
## ドキュメント整合性チェック結果

### OK（整合）
- ✅ ...

### NG（不整合）
- ❌ [ファイル名]: 問題の説明
  - 現在の記述: `...`
  - 実際の状態: `...`
  - 修正案: `...`

### 要確認
- ⚠️ ...
```

不整合が見つかった場合は、修正するかどうかを確認してから編集してください。
